## Bug Reports

For public, non-sensitive bugs, open a GitHub issue and include:

- what command you used
- what you expected
- what happened instead
- whether the issue happens every time or only sometimes
- screenshots with private information removed, if helpful

Do not post Discord bot tokens, private Discord tokens, passwords, account credentials, or private account information.

## Feature Requests

Feature requests are welcome when they are safe to discuss publicly. Good requests explain:

- the workflow you are trying to improve
- what command or screen is involved
- what would make the bot faster or clearer

## Sensitive Issues

If a report includes credentials, private account details, security concerns, or anything that should not be public, do not open a public GitHub issue. Use the support Discord or email instead, and keep the report high-level until a private channel is established.
