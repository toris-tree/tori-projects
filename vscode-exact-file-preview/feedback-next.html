What file type or workflow should Exact File Preview support next?

It previews the exact selected file in VS Code without starting a server or letting a framework/router decide what gets served.

Current shape

Exact File Preview supports HTML, SVG, Markdown, JSON, XML, and plain text in a sandboxed side panel. It is intentionally local and read-only: no server, no telemetry, scripts blocked.

What I am looking for

I am trying to learn where exact local preview is useful enough to keep installed.

  • What file type should this support next?
  • What workflow makes a read-only exact preview more useful than Live Server or a framework preview?
  • Do docs, config, generated artifacts, SVG/design review, or team review flows need different behavior?
